01 / Overview

From desktop idea to reproducible system

MARISHOKU/OS explores how far a cohesive desktop concept can be carried through a real Debian-based build. It combines Plasma theming, custom tools, OMOTE and URA profiles, a Debian package and a repeatable live-build configuration for producing a hybrid ISO.

01

Boot-to-desktop flow

Branding, session defaults and the desktop shell are treated as one continuous experience rather than disconnected themes.

02

Custom desktop layer

Profiles, tooling and Plasma configuration shape the working environment while retaining Debian's package foundation.

03

Reproducible builds

Package and ISO automation make the experiment repeatable instead of depending on one hand-configured machine.

02 / Build decisions

A concept that can actually be rebuilt

The project separates visual assets, desktop defaults, package contents and live-build configuration. That makes changes traceable and allows a fresh image to be produced from source rather than copied from an existing installation.
